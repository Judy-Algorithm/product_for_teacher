# 端侧便携 AI 阅卷仪后续优化路线

版本：v0.1  
日期：2026-06-04

## 1. 文档目的

本文记录暂不进入 MVP、但后续可显著提升成本效率、批改准确率和产品价值的优化方向。

当前 MVP 优先保证核心闭环跑通：稳定拍摄、试卷裁剪、题目定位、手写识别、题型分流、规则判分、大模型评分、老师校准、结果回写。

## 2. 成本优化方向

### 2.1 后端小模型初筛

后续可在后端引入轻量模型或规则模型，对部分计算题、填空题和短答案题进行初筛。

目标：

- 高置信度题目直接给分。
- 中低置信度题目再调用大模型。
- 降低大模型 API 调用次数。
- 缩短批改耗时。

适用题型：

- 简单计算题。
- 数值型填空题。
- 固定答案短文本题。
- 部分步骤较清晰的数学题。

### 2.2 评分标准结构化

当前阶段可允许老师以自然语言录入评分标准，降低研发投入和使用门槛。

后续可将评分标准自动转换为结构化规则，例如：

```json
{
  "question_no": "3",
  "full_score": 5,
  "points": [
    {"name": "公式正确", "score": 2},
    {"name": "代入正确", "score": 1},
    {"name": "计算结果正确", "score": 2}
  ],
  "deduction_rules": [
    {"condition": "结果错误但过程正确", "deduct": 2},
    {"condition": "缺少单位", "deduct": 0.5}
  ]
}
```

价值：

- 减少 Prompt token。
- 提升大模型输出稳定性。
- 方便规则判分和大模型评分共用同一套标准。
- 方便后续做题型模板和评分标准复用。

### 2.3 批量调用优化

后续可根据题目类型和答卷数量进行请求合并。

优化方式：

- 同一道题的多个学生答案批量送入一次请求。
- 同一学生的多个短题合并请求。
- 大模型只处理规则无法判定的题。
- 对已稳定的评分规则生成短 Prompt 摘要。

## 3. 准确率优化方向

### 3.1 校准样本增强

当前 MVP 采用前 3-5 张试卷作为本次任务校准样本。

后续可增强为：

- 系统主动挑选最有代表性的样本让老师校准。
- 优先让老师修正规则判分置信度低、大模型判断不确定的题。
- 将校准样本按题型、知识点、错误类型归档。

### 3.2 个性化评分习惯

在积累足够老师修正数据后，可按老师、学科、年级和题型建立评分偏好。

可优化内容：

- 语文简答题宽严程度。
- 英语作文评分习惯。
- 数学步骤分给分口径。
- 填空题等价答案识别。

首期不建议直接做长期个性化模型，避免研发复杂度过高。

## 4. 教学分析能力

### 4.1 班级答题情况总结

批改完成后，系统可自动总结全班答题情况。

输出内容：

- 每道题正确率。
- 高错题列表。
- 常见错误类型。
- 班级平均分。
- 分数分布。
- 需要重点讲解的知识点。

价值：

- 帮助老师快速备讲评课。
- 将产品从“批改工具”升级为“教学反馈工具”。

### 4.2 知识点掌握分析

后续可将题目与知识点绑定，批改后输出知识点掌握情况。

可包括：

- 班级未掌握知识点。
- 学生个人薄弱知识点。
- 高频错误知识点。
- 建议复习顺序。

该能力需要题目知识点标注，可先由老师手动选择，后续再尝试自动识别。

### 4.3 学生档案卡

为每个学生建立长期学习档案卡。

记录内容：

- 每次作业和测验成绩。
- 每题得分情况。
- 错误知识点。
- 重复出错题型。
- 常见答题习惯问题。
- 老师评语记录。

可分析：

- 哪些知识点反复理解错误。
- 哪些答题习惯反复扣分。
- 成绩是否稳定提升。
- 哪类题型需要针对性训练。

## 5. 产品功能优化

### 5.1 错题本

自动汇总学生错题，支持按知识点、题型、错误原因筛选。

### 5.2 讲评课助手

基于全班高频错题生成讲评课清单。

### 5.3 评分模板库

沉淀常用题型评分模板，减少老师每次录入评分标准的时间。

### 5.4 电子批改图样式优化

支持不同展示模式：

- 只显示分数。
- 显示分数和简短原因。
- 显示详细批改说明。
- 学生版和老师版分开展示。

## 6. 优先级建议

第一阶段：

- 后端小模型初筛。
- 评分标准结构化。
- 批量调用优化。

第二阶段：

- 班级答题情况总结。
- 高错题统计。
- 知识点手动绑定。

第三阶段：

- 学生档案卡。
- 长期个性化评分习惯。
- 自动知识点识别。
