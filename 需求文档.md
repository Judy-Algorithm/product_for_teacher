# 端侧便携 AI 阅卷仪需求文档

版本：v0.1  
日期：2026-06-04

## 1. 项目背景

中小学教师在日常教学中需要频繁批改作业、随堂测、小测卷和自编练习。传统人工批改耗时长，纯手机 APP 拍摄不稳定，重型阅卷机价格高、体积大、偏学校采购，不适合个人老师。

本项目拟开发一款便携式 AI 阅卷仪，通过低成本硬件采集高清作业图像，结合自研图像预处理、题目定位、手写识别和云端通用大模型评分，为老师提供低成本、高效率、可解释的批改辅助工具。

## 2. 产品目标

首期目标：

- 让老师能够快速拍摄答案卷和学生答卷。
- 支持多形态、多介质纸质作答载体，包括自编卷、练习册、作业页和小测卷，不强制使用固定答题卡。
- 支持老师输入给分点和评分标准。
- 将学生答题内容识别为文本后交给大模型批改。
- 对选择题、判断题和简单填空题优先通过规则比对判分，减少大模型调用。
- 输出每题得分、扣分原因和电子批改图。
- 支持老师人工修正 AI 结果，并用前 3-5 份样本校准后续批改。

非首期目标：

- 不做高利害考试正式阅卷。
- 不做本地大模型推理硬件。
- 不做完整学校教务系统。
- 不做复杂统考级保密和审计系统。

## 3. 目标用户

核心用户：

- 小学、初中、高中任课老师。
- 培训机构老师。
- 民办学校教师。
- 需要批改日常作业、小测卷、自编卷的个人教师。

典型用户场景：

- 老师课后批改一班 40-50 份数学小测。
- 英语老师批改填空、简答和作文练习。
- 语文老师批改阅读理解和作文初稿。
- 培训机构老师批改多名学生随堂练习。
- 老师批改非标准答题卡的教辅练习册。

## 4. 核心用户流程

### 4.1 新建批改任务

老师打开 APP，创建一次作业或测验任务，填写任务名称、班级、科目、年级和总分。

### 4.2 拍摄答案卷

老师将答案卷放在黑色底座板上，通过设备 Head 摄像头拍摄答案卷。

系统自动裁剪试卷区域，并提示老师确认图片是否清晰。

### 4.3 设置评分标准

老师为每道题设置：

- 题号。
- 分值。
- 标准答案。
- 给分点。
- 扣分规则。
- 可选评语口径。

老师也可以根据系统引导，用红笔在答案卷上做简单区域标记，辅助系统识别每道题的位置。

### 4.4 扫描学生答卷

老师逐张将学生答卷放在底座上拍摄。

系统每拍一张，自动完成：

- 试卷区域裁剪。
- 透视矫正。
- 去阴影。
- 题目区域定位。
- 手写内容识别。
- 题型分流。
- 规则判分或 AI 批改评分。
- 电子批改图生成。

### 4.5 老师确认和修正

APP 展示 AI 批改结果。

老师可执行：

- 修改分数。
- 修改评语。
- 标记 AI 判错原因。
- 确认本张答卷批改完成。

前 3-5 张试卷的人工修正会作为本次任务的校准样本，帮助后续答卷批改更贴合老师评分标准。

### 4.6 输出结果

系统输出：

- 原卷电子批改图。
- 每题得分。
- 总分。
- 扣分理由。
- 错因记录。
- 成绩表。

老师可根据电子批改图在纸质试卷上手写分数，也可将电子版发送给学生。

## 5. 功能需求

### 5.1 硬件功能

P0：

- 支持折叠收纳。
- 支持摄像头自动对焦。
- 支持拍摄 A4、B5、练习册、作业页等常见纸质载体。
- 支持 Head 与底座通过可伸缩支架杆连接。
- 支持通过支架高度调节拍摄范围。
- 支持黑色底座板辅助识别纸张边界。
- 支持 Type-C 摄像头直连手机或平板，由 APP 读取画面并上传云端。

P1：

- 支持 WiFi 摄像头版本，提供无线拍摄体验。
- 支持补光灯。
- 支持设备 SN 绑定账号。
- 支持拍摄状态提示。
- 支持异常拍摄提醒，如模糊、倾斜、遮挡、光线不足。

P2：

- 支持多档高度定位。
- 支持离线缓存图片后联网补传。
- 支持更大幅面纸张拍摄。

### 5.2 APP 功能

P0：

- 登录和账号管理。
- 设备绑定。
- 新建批改任务。
- 拍摄答案卷。
- 设置题号、分值、答案和评分标准。
- 拍摄学生答卷。
- 查看 AI 批改结果。
- 修改 AI 分数。
- 修改 AI 评语。
- 标记 AI 判错原因。
- 查看电子批改图。
- 导出成绩表。

P1：

- 批量连续扫描。
- 班级管理。
- 学生名单管理。
- 任务历史记录。
- 错题记录。
- 评分标准模板复用。
- 常用评语模板。

P2：

- 多老师协作。
- 学情报告。
- 家长或学生端电子批改图查看。
- 会员权益管理。
- 多套评分口径版本管理。

### 5.3 图像预处理功能

P0：

- 基于黑色底板进行纸张区域检测。
- 自动裁剪试卷或作业范围。
- 自动旋转矫正。
- 自动透视矫正。
- 基础去阴影。
- 输出标准化答卷图片。

P1：

- 模糊检测。
- 过曝或欠曝检测。
- 边角缺失检测。
- 装订类纸质载体弯曲矫正。
- 多规格纸张自动识别。

P2：

- 复杂背景鲁棒识别。
- 多页连续识别。
- 练习册跨页拍摄。

### 5.4 题目定位功能

P0：

- 基于答案卷建立题目位置模板。
- 支持老师用红笔或简单标记辅助划分题目区域。
- 支持网格算法自动划分答题范围。
- 支持将答案卷题目区域映射到学生卷。

P1：

- 支持老师手动调整题目框。
- 支持保存题目框模板。
- 支持不同试卷版式复用。

P2：

- 支持自动识别题号。
- 支持多栏试卷。
- 支持复杂排版和跨页题目。

### 5.5 手写识别功能

P0：

- 识别选择、判断、填空答案。
- 识别中文、英文、数字和常见符号。
- 将每道题学生答题内容转换为文本。
- 将识别文本与题号、题目框位置绑定。

P1：

- 支持数学公式和计算步骤识别。
- 支持潦草手写置信度提示。
- 支持老师查看识别文本并修正。

P2：

- 支持更多公式结构化表达。
- 支持作文段落识别和错别字标记。

### 5.6 AI 批改功能

P0：

- 支持题型分流。
- 支持选择题、判断题直接规则比对判分。
- 支持简单填空题通过 OCR 文本和标准答案比对判分。
- 基于老师输入的标准答案和评分标准进行评分。
- 支持按题输出得分。
- 支持输出扣分原因。
- 支持输出简短评语。
- 支持输出错误类型。
- 支持前 3-5 份老师修正反馈影响后续评分。

P1：

- 支持分题型 Prompt 模板。
- 支持数学步骤分。
- 支持语文、英语简答题踩点评分。
- 支持作文基础评分。

P2：

- 支持个性化评分习惯。
- 支持按老师、学科、题型沉淀评分策略。
- 支持轻量微调或长期 Prompt 优化。

### 5.7 结果回写功能

P0：

- 将每题得分插入原答卷图片对应位置。
- 将简短扣分理由插入对应题目附近。
- 生成电子批改图。
- 支持老师查看和确认。

P1：

- 支持标注样式调整。
- 支持只显示分数、显示分数+理由两种模式。
- 支持导出图片或 PDF。

P2：

- 支持学生端查看。
- 支持电子错题本自动生成。

## 6. 技术架构需求

### 6.1 架构分层

系统分为五层：

1. 硬件采集层：摄像头、通信模块、支架、底座。
2. APP 交互层：任务创建、拍摄、设置评分标准、结果查看和修正。
3. 图像算法层：裁剪、矫正、定位、切题框。
4. 识别与结构化层：手写 OCR、题目文本结构化。
5. 批改决策层：题型分流、规则判分、大模型调用、评分、理由生成、结果回写。

### 6.2 大模型调用策略

系统不直接将整张答卷图片交给大模型批改。系统应先完成题型分流：

- 选择题、判断题：通过标准答案直接比对。
- 简单填空题：通过 OCR 文本与标准答案比对，可支持大小写、空格、简单同义词或数值误差规则。
- 复杂题型：进入大模型评分链路。

这样可以显著减少大模型调用次数，降低批改成本，并提高基础题型评分稳定性。

推荐输入方式：

```json
{
  "task_id": "任务 ID",
  "question_id": "题号",
  "score": "本题分值",
  "standard_answer": "标准答案",
  "grading_rules": "老师输入的给分点和扣分规则",
  "student_answer_text": "手写识别后的学生答案文本",
  "calibration_examples": "老师前几份修正样本"
}
```

推荐输出方式：

```json
{
  "question_id": "题号",
  "score_awarded": "实际得分",
  "deduction_reason": "扣分原因",
  "grading_basis": "给分依据",
  "error_type": "错误类型",
  "student_feedback": "可展示给学生的简短评语"
}
```

## 8. 验收指标

硬件与拍摄：

- A4 纸张完整拍摄成功率不低于 95%。
- 自动对焦成功率不低于 95%。
- 常规教室光线下图片清晰可读。
- 折叠和展开操作简单稳定。

图像算法：

- 基于黑色底板的纸张裁剪成功率不低于 95%。
- 旋转和透视矫正后文字可读。
- 答案卷模板到学生卷题目框映射成功率不低于 90%。

识别与批改：

- 客观题批改准确率不低于 95%。
- 常规填空题识别和批改准确率不低于 85%。
- 选择题、判断题和简单填空题优先不调用大模型。
- 主观题评分需要提供依据，并允许老师快速修正。
- 单题批改结果必须包含分数和理由。

效率：

- 单张答卷从拍摄到返回初步结果目标控制在 30 秒以内。
- 支持一班 40-50 份答卷连续处理。
- 老师修正一题的操作不超过 3 步。

输出：

- 电子批改图中分数和理由位置清晰，不遮挡关键答题内容。
- 成绩表可导出。
- 老师可追溯每题 AI 判分理由。

## 9. 风险与应对

### 9.1 手写识别准确率不足

应对：

- 提供识别置信度。
- 允许老师修正识别文本。
- 优先支持高频题型，再逐步扩展复杂题型。

### 9.2 大模型评分不稳定

应对：

- 不让模型自由评分。
- 强制基于评分标准输出。
- 引入前几份老师修正样本作为校准。
- 按题型使用固定 Prompt 模板。

### 9.3 拍摄环境复杂

应对：

- 使用黑色底座板增强纸张边界。
- 增加补光。
- APP 提示模糊、阴影、遮挡等异常。

### 9.4 API 成本过高

应对：

- 避免整图直接输入大模型。
- 先 OCR，再以文本 token 输入。
- 按题拆分和合并调用策略优化。
- 图片仅用于算法定位和必要复核。

### 9.5 用户不愿录入评分标准

应对：

- 支持答案卷拍摄和自动解析。
- 支持评分模板复用。
- 支持老师只录入关键给分点。
- 优化首次任务创建流程。

## 10. MVP 范围

MVP 必须完成：

- 可折叠拍摄硬件样机。
- 黑色底板辅助裁剪。
- 答案卷拍摄。
- 老师录入评分标准。
- 学生答卷逐张拍摄。
- 题目区域定位。
- 手写 OCR 基础能力。
- 题型分流。
- 选择题、判断题和简单填空题规则判分。
- 文本输入大模型评分。
- 分数和理由回写电子答卷图。
- 老师改分和改评语。
- 成绩导出。

MVP 可暂缓：

- 多老师协作。
- 学生端。
- 复杂学情报告。
- 长期个性化模型微调。
- 高利害考试合规能力。

## 11. 一句话总结

本产品的核心不是“摄像头 + 大模型”，而是“稳定拍摄 + 试卷结构化 + 老师评分标准 + 手写文本化 + 低成本大模型评分 + 老师反馈校准”的完整批改闭环。
