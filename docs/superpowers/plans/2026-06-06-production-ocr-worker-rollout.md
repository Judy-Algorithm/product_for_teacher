# Production OCR Worker Rollout Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the teacher grading app run end-to-end in production with Vercel, Vercel Blob, Neon Postgres, and a Dockerized Python OpenCV/PaddleOCR worker.

**Architecture:** Vercel hosts the Next.js UI and lightweight API routes. Vercel Blob stores original papers and crop images, Neon Postgres stores structured job/rubric/result/report data, and a separate Python worker service performs OpenCV crop correction plus PaddleOCR before calling back into Vercel.

**Tech Stack:** Next.js App Router, Vercel, Vercel Blob, Neon Postgres, FastAPI, Docker, OpenCV, PaddleOCR, Kimi API.
